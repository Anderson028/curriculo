Eficiência do Request HTTP

A eficiência do Request HTTP se dá por alguns fatores como as tamanho e quantidade solicitações e respostas, ou seja quantas solicitações que estão sendo solicitada, e as que estão sendo requisitadas em paralelo. Cache é quando o navegador ele armazena o site, e caso aquele site não tenha tido uma atualização, o navegador carrega sem requisitar o servidor do site que acessou, sendo assim uma espécie de "Checkpoint". Compactação de dados o HTTP pode comprimir dados antes mesmo de enviar facilitando e deixando mais leve seu envio. Latência de rede é a conexão entre o cliente e o servidor sendo assim caso o cliente tenha uma internet ruim ele não vai ter um acesso eficiente e pode gerar problemas de perfomace.
